为什么路由器获取动态IP失败？

发表于 2018-10-27 | 分类于 网络

最近由于网络不稳定，所以请物业过来改造了一下网络结构

改造前的网络拓扑图：

  • 光猫
    • 公寓路由器（公用WIFI）
      • 家庭路由器1
      • 家庭路由器2
      • 家庭路由器n…

改造后的网络拓扑图：

  • 光猫
    • 公寓路由器（公用WIFI）
    • 家庭路由器1
    • 家庭路由器2
    • 家庭路由器n…

相当于房间的网线直接连入到光猫中（因为房间比较少，所以端口够用），但是这时之前能够动态获取IP的家庭路由器获取不到IP了

Q：测试入室网线是否连通
A：直接连接网线到电脑，可以动态获取IP，证明网线没有问题

Q：把上一步获取的IP、网关等参数记录下来，使路由器以静态IP方式入网
A：路由器不能联通，但出现以下提示。

这个时候问题已经很明确了，因为光猫的IP和家庭路由器的IP冲突了

解决办法：

  1. 登陆家庭路由器管理页面
  2. 依此选择网络参数 > LAN口设置，不同品牌的路由器设置大同小异
  3. 修改IP地址为192.168.xxx.1，这里的IP可以是与光猫IP不同的任意一个数字，比如192.168.5.1
  4. 保存，搞定！

git repo简介

发表于 2018-10-27 | 分类于 学习

参考：

  • http://www.360doc.com/content/14/0220/17/97538_354256755.shtml
  • http://blog.csdn.net/sunweizhong1024/article/details/8055372

repo init做了什么？

repo init <-u url> [--repo-url=url] [-m default.xml] [-b master]

参数 含义 缺省
–repo-url repo 工具本身的 git 库地址 git://android.git.kernel.org/tools/repo.git
-u manifest仓库的URL 必须指定
-m 在manifest仓库中选择一个xml文件 default.xml
-b 选择一个manifest仓库中一个指定的分支 master
  • 完成repo工具的下载，执行repo脚本（–repo-url）
  • 克隆清单库到本地（-u）
  • .repo文件夹下生成软链接manifest.xml指向.repo/manifests/default.xml（-m）

repo sync做了什么？

repo sync [<project> ...]

  • 参照清单.repo/manifest.xml克隆并同步版本库
    • 版本库不存在：git clone
    • 版本库存在：git remote update 和 git rebase origin/branch

总结：repo相当于对Git命令的封装
